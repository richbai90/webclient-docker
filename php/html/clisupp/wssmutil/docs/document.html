The WSS Wizard Import/Export Utility User Guide

Introduction

The WSS Wizard Import/Export Utility can be used to move WSS Wizards between Supportworks databases, using wizard data stored as XML files.

This utility is launched from within the File > Manage Settings > General Settings > SelfService Wizards tab of the Supportworks Client - see the Supportworks ITSM Enterprise Administration Guide for more information.

Functions available

Utility functions available include:

  • Export: Export one or more wizards from Supportworks, generating an XML file
  • Manage Files: upload, download or delete generated XML files
  • Import: Import one or more generated XML files into Supportworks as wizards.

Prerequisites

You will need an ODBC connection between the server running the utility, and any system you want to import from or export to.

Export

The Export option provides a simple way to transfer data between Supportworks databases.

It allows users to export single, multiple, or all wizards in a database into XML format.

For example:

Select the wizard(s) you want to export from the list, then select Export Wizard to generate XML files - one file for each wizard.

Data Source: the data sources that the system you are connected to has been configured for. The Use Default DSN for Server option is default DSN that Supportworks is configured to use.

Manage Files

This option shows all generated XML files which are currently available for use for the Utility.

For example:

You can download or delete these files.

To upload new files into the utility (for use to import into Supportworks), select the file using the Browse button, then click Upload File.

(Note: only XML files can be accepted for upload)

Import

This option provides users with the option to import generated XML files of Wizards into a Supportworks database.

For example:

Select the wizard file(s) required, then select Import Wizard to import that Wizard or Wizards into the Supportworks system.

Generation of primary keys

During the import new primary keys will be created for some tables (For example, pk_auto_id within the wssm_wiz_q table will be automatically generated).

These automatically generated primary keys exist as foreign keys in related tables.

During the import these foreign keys will be updated with the values of the newly generated primary keys.
